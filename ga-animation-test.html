Text Line
Clip reveal, line by line

Each line of a headline clips upward into view. Uses overflow:hidden on the container with translateY(100%→0) per span. Editorial-grade.

Marketing isn't about moving people through stages. It's about reading transitions.
Implementation Note
Each <span> must be a block element. Parent needs overflow:hidden. WP.com strips inline styles from page content — these spans need classes in style.css, not inline styles. Test this in the block editor.

Zone 08 · Parallax Drift pending

Background Parallax
Scroll-synced depth

Background layer moves at 0.4× scroll speed relative to container. Pure JS — no CSS scroll-snap. Check for jank on lower-end devices. Uses requestAnimationFrame.

Depth in Motion

background moves at 0.4× scroll velocity

Performance Note
Parallax is driven by requestAnimationFrame + transform: translateY(). Will-change: transform is set on the bg element. Avoid parallax on mobile — the scroll behavior differences cause jank. Add a @media (prefers-reduced-motion: reduce) kill-switch.

Summary
Every animation above uses a single shared IntersectionObserver (threshold tunable above). The stagger, count-up, line-reveal, parallax, and border-draw each add one small behavior on top of the same IO pattern. This is the architecture to carry into ga-animations.js — one observer, multiple callbacks keyed by data-anim attribute.
